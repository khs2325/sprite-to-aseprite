# OpenRaster `.ora` Format Notes

Status: researched subset for future importer work. No OpenRaster importer or UI
wiring is implemented by this task.

Primary references:

- OpenRaster 0.0.6 file layout: https://www.openraster.org/baseline/file-layout-spec.html
- OpenRaster 0.0.6 layer stack: https://www.openraster.org/baseline/layer-stack-spec.html
- OpenRaster animation proposal: https://www.openraster.org/proposals/animation.html
- OpenRaster filter effects proposal: https://www.openraster.org/proposals/filter-effects.html

OpenRaster is a ZIP container for layered raster documents. The baseline file
layout defines these standard entries:

- `mimetype`: the first ZIP entry, stored without compression, exactly
  `image/openraster` with no trailing whitespace.
- `stack.xml`: UTF-8 XML describing canvas size and layer stack metadata.
- `data/`: layer image data referenced by `stack.xml`, typically PNG files.
- `Thumbnails/thumbnail.png`: a browser/file-manager preview, not source layer
  data.
- `mergedimage.png`: the final flattened preview image, not source layer data.

The ZIP member names are case-sensitive and should be UTF-8. Only `STORED` and
`DEFLATED` entries are part of the safe subset. The importer must read all user
files locally in the browser.

## Supported Subset

The first importer should accept only single-frame layered raster images that
can be represented directly as `SpriteProject`:

```ts
{
  width: image.w,
  height: image.h,
  colorMode: "rgba",
  frames: [{ index: 0, durationMs: 100 }],
  layers: [
    {
      id,
      name,
      visible,
      opacity,
      cels: [{ frameIndex: 0, x, y, imageData }],
    },
  ],
}
```

Accepted `stack.xml` shape:

- Root `<image>` with positive integer `w` and `h`.
- Exactly one root `<stack>`.
- Root stack contains only direct `<layer>` children. OpenRaster defines the
  first child in a stack as the uppermost layer; importer tests should preserve
  and assert that source order.
- Each `<layer>` has a `src` attribute pointing to a PNG file inside the ZIP,
  normally under `data/`.
- Layer PNG data decodes to RGBA pixels in the browser.
- Optional `name` is preserved. If omitted, use a deterministic generated name
  such as `Layer 1`.
- Optional `x` and `y` are signed integer cel offsets. Defaults are `0`.
- Optional `visibility` is `visible` or `hidden`. Default is `visible`.
- Optional `opacity` is a number from `0.0` through `1.0`. Default is `1.0`.
- Optional `composite-op` is either omitted or `svg:src-over`. This is the
  normal alpha-compositing mode.

Layer paths must be normalized relative paths inside the archive. Reject
absolute paths, path traversal (`..`), backslash-separated paths, empty paths,
directories, and external references. Do not decode or use `mergedimage.png` to
recover layers; it is only a flattened preview.

## Rejections And Unsupported Features

Reject the entire file with a clear diagnostic for:

- Missing or incorrect `mimetype`.
- Missing `stack.xml`, malformed XML, or a missing/invalid root `<image>`.
- Missing, unreadable, non-PNG, or corrupt PNG data referenced by a layer.
- Non-positive canvas dimensions, non-integer `x`/`y`, opacity outside `0..1`,
  or unknown `visibility` values.
- Any unsupported `composite-op`, including valid OpenRaster modes such as
  `svg:multiply`, until blend-mode behavior is explicitly implemented and
  tested.
- Nested `<stack>` elements, groups, isolation semantics, text/vector layers,
  masks, selections, adjustment/filter/effect layers, or unknown layer-like
  elements under the root stack.
- Animation or multiple-page metadata. OpenRaster animation is proposal-stage
  metadata, not part of this single-frame subset.

Ignore archive entries that are not referenced by the supported subset, such as
extra metadata files, as long as required conversion data is valid. Ignore
`xres` and `yres` for `SpriteProject` construction because the current internal
model has no resolution field.

## Fixture Strategy

Fixtures are generated by `tests/fixtures/generate.mjs` and contain only
synthetic pixels:

- `tests/fixtures/openraster/two-layers.ora` is the positive fixture. It has a
  valid `mimetype`, `stack.xml`, two PNG-backed raster layers, layer names,
  source order, explicit and default visibility, explicit and default opacity,
  signed x/y offsets, explicit `svg:src-over`, thumbnail, and merged preview.
- `tests/fixtures/openraster/unsupported-blend-mode.ora` changes one layer to
  `svg:multiply` so the importer task can reject unsupported blend modes.
- `tests/fixtures/openraster/missing-layer-data.ora` references a layer PNG that
  is intentionally absent from the ZIP.
- `tests/fixtures/openraster/malformed-stack.ora` contains malformed
  `stack.xml` while keeping the ZIP and PNG data deterministic.

Fixture tests inspect ZIP signatures, `mimetype`, `stack.xml` metadata, PNG
signatures, and expected structural metadata. They do not implement OpenRaster
conversion.
